Database Schema

InternHive Database Structure and Relationships

Database Tables

Complete schema for the InternHive system

Admins Table

  • id: UUID (Primary Key)
  • name: VARCHAR(255) NOT NULL
  • email: VARCHAR(255) UNIQUE NOT NULL
  • role: VARCHAR(100) NOT NULL
  • password_hash: VARCHAR(255) NOT NULL
  • created_at: TIMESTAMP DEFAULT NOW()
  • updated_at: TIMESTAMP DEFAULT NOW()

Interns Table

  • id: UUID (Primary Key)
  • name: VARCHAR(255) NOT NULL
  • email: VARCHAR(255) UNIQUE NOT NULL
  • phone: VARCHAR(20)
  • university: VARCHAR(255)
  • program: VARCHAR(255)
  • skills: TEXT[]
  • password_hash: VARCHAR(255) NOT NULL
  • status: VARCHAR(50) DEFAULT 'active'
  • start_date: DATE
  • end_date: DATE
  • created_at: TIMESTAMP DEFAULT NOW()
  • updated_at: TIMESTAMP DEFAULT NOW()

Tasks Table

  • id: UUID (Primary Key)
  • title: VARCHAR(255) NOT NULL
  • description: TEXT
  • assigned_to: UUID (Foreign Key → interns.id)
  • assigned_by: UUID (Foreign Key → admins.id)
  • status: VARCHAR(50) DEFAULT 'pending'
  • priority: VARCHAR(20) DEFAULT 'medium'
  • due_date: TIMESTAMP
  • completed_at: TIMESTAMP
  • created_at: TIMESTAMP DEFAULT NOW()
  • updated_at: TIMESTAMP DEFAULT NOW()

News Table

  • id: UUID (Primary Key)
  • title: VARCHAR(255) NOT NULL
  • content: TEXT NOT NULL
  • author_id: UUID (Foreign Key → admins.id)
  • published: BOOLEAN DEFAULT false
  • created_at: TIMESTAMP DEFAULT NOW()
  • updated_at: TIMESTAMP DEFAULT NOW()

Attendance Table

  • id: UUID (Primary Key)
  • intern_id: UUID (Foreign Key → interns.id)
  • check_in: TIMESTAMP
  • check_out: TIMESTAMP
  • date: DATE NOT NULL
  • status: VARCHAR(20) DEFAULT 'present'
  • notes: TEXT
  • created_at: TIMESTAMP DEFAULT NOW()

Setup Instructions

How to connect to Supabase

  1. Create a new project at supabase.com
  2. Copy the project URL and anon key
  3. Update the configuration in supabase-client.js
  4. Run the SQL schema from supabase-schema.sql
  5. Enable Row Level Security on all tables
